JSON

JSON(javascript Object Notation) : 자바스크립트 객체 표기법

자바스크립트 객체 = 모든 자료형

        JSON을 사용하면 모든 데이터형을 서버와 주고받을 수 있다. (사용목적)

        숫자, 문자열, boolen, 배열, 객체, null (종류)


        ▶ 자바스크립트 표기법

        중괄호'{'으로 시작하여 '}'로 끝나며 그 속에 데이터를 표기하고 'key : value(값)' 쌍으로 구성된다.

        {
            "name" : "이순신",
            "age" : 27,
            "birth" : "1990-01-01",
            "gender" : "남",
            "marry" : true,
            "address" : "서울특별시 중구 인사동",
            "family" : {  
                        "father" : "아버지",
                        "mother" : "어머니",
                        "brother" : "동생"
                    }
        }

        var student = new Object(); => {}
        var arrs = new Array(); => []
        var num = new Number(3); => 3
        과 같이 간단히 표기하는 방법이 "JSON"이다.
구분값 Javascript Object Javascript Object Notation(JSON)
Boolean var n = new Boolean(true); var n = true;
Number var n = new Number(3); var n = 3;
String var s = new String("hello"); var s = "hello";
Array var arrs = new Array(); var arrs = [];
Object var obj = new Object(); var obj = {};



대표적인 데이터를 주고받는 형태 Top3

XML, CSV, JSON

1. XML 방식 : notice 작성이 가능 ( notices태그를 통해 notice 전달을 알리고, notice 태그에 정보들을 담아 전달 )
- 단점 : 용량이 크며, 꺽쇠를 사용해야하는 불편함 존재

2. CSV 방식 : 데이터(정보)를 콤마(,)로 구분하여 전달 ( id="1" title="All the json" 등의 정보를 콤마(,)로 구분)
- 단점 : 작성에 용이하지만, 데이터를 구분하기 어렵다는 불편함 존재
- (글자가 깨지거나, 띄어쓰기로 인한 구분 오류로 정확한 데이터 전달이 불가능할 가능성

1, All the json
2, json class
3, more about json



3. JSON방식 : XML과 CSV의 장점을 합쳐놓은 방식 ( 구조 : [{:}] -> 대괄호/중괄호/콜론으로 연결 )
- 여러개의 데이터를 보낼 경우 : 배열로 전달 []
- 전달할 데이터 : 중괄호 {키:값}
- 데이터 구분 : 콤마사용 [ {}, {}, {} ]

    { "id" : 1, "title" : "All that json" },
